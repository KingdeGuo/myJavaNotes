# Java语法部分基础知识复习

- Java1.5之后的三大版本

  - Java SE 标准版
  - Java EE 企业版
  - Java ME（J2EE） 微型版

- 采用字节码的好处

  在一定程度上解决了传统解释型语言执行效率低的问题，同时又保留了解释性语言可移植的优点。

- 编译程序只需面向虚拟机，生成虚拟机能够理解的代码，然后由解释器来讲代码转化为特定系统的机器代码执行。

- 应用程序和小程序的主类有何不同

  应用程序的主类不一定是`public`的，但是小程序的主类一定是`public`的。

  应用程序是从主线程启动（也就是`main()`方法），applet小程序没有`main()`方法，主要是嵌在浏览器页面上运行（调用`init()`线程或者`run()`来启动）。

- Java和C++的区别与联系

  - 都是面向对象的，都支持封装、继承、多态。
  - Java不支持直接使用指针来访问内存，程序使用内存更加安全。
  - Java的类是单继承的，C++支持多重继承，虽然Java的类不可以多继承，但是接口可以多继承。
  - Java由自动管理内存机制，不需要程序员手动释放无用内存。

- Oracle JDK 与 Open JDK的区别

  - Oracle JDK每三年发布一次，OpenJDK每三个月发布一次。
  - OpenJDK是一个参考模型并且是完全开源的，Oracle JDK的Open JDK的一个实现，不是完全开源的。
  - Oracle JDK比Open JDK更加稳定。
  - 在响应性和VM性能方面，Oracle JDK提供了更好的性能。
  - Oracle JDK根据二进制代码许可协议获得许可，而Open JDK根据GPL v2许可获得许可。

- 关于`switch`

  ​	注意，不支持`long`

  - 在JDK1.5之前，`switch(expr)`中，expr只能是`byte`，`short`, `char`, `int`
  - 在JDK1.5中引入了枚举。
  - 在JDK1.7中引入了字符串。

- 下面代码有错吗

  ```java
  short short1 = 1;
  short1+=1;
  System.out.println(short1);
  ```

  注意在执行`short1 += short1`时，`short1`会提升为`int`型，所以结果也是`int`型，需要强制转化才能转化成`short`型。

  注意：上面是可以编译通过，因为`short1 += short1`相当于`short1 = (short)(short1+1)`，其中隐含了强制类型转化。

- `final`的作用

  - 修饰类：类不可继承

  - 修饰字段：被修饰的变量不可以被改变，被final修饰不可变的是变量的引用，而不是指向变量的内容，引用指向的内容是可以改变的

    ```
    final Student s = stu;
    // 这个例子中，s将只能指向stu，这个不可以改变
    // stu的状态可以改变，例如sru.age = 19，这个是可以改变的。
    ```

    

  - 修饰方法：方法不可被重写

- `this`的作用

  - 普通的直接引用，`this`相当于指向当前对象本身
  - 形参与成员名字重名
  - 引用本类构造器

- `super`的作用

  - 普通的直接引用，`super`相当于指向当前对象的父类的引用。
  - 子类的成员变量**或方法**与父类中的相同
  - 引用父类构造器

- `this`和`super`不能出现在同一个构造函数里面。

- 从本质上来讲，`this`是一个指向本对象的指针，`super`是一个Java关键字。

- `static`的主要作用

  - 创建独立于具体对象的域变量或方法
  - 用来行成静态代码块以优化程序性能
