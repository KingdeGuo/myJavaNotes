# 分布式

- 两大设计思路
  - 中心化设计
    - Master负责监督与任务分配。
    - 为了避免Master故障可以同时安排多个Master。
    - 要注意Master的“领导”能力，当其可以领导100台机器时，并不意味着扩展到1000台机器时没有问题。
  - 去中心化设计
    - 不是不要中心，而是自由选择中心。
    - 尤其需要注意避免脑裂问题，解决方式一般是规模较小的集群关闭服务。
    - 全球互连网就是一个去中心化的分布式系统。
- CAP
  - 一致性
  - 可用性
  - 分区容错性
- BASE
  - 基本可用：分布式系统出现不可预知的故障时，允许损失部分性能。
    - 响应时间上损失
    - 系统功能上损失
  - 软状态：系统中的数据存在中间状态
  - 最终一致性：系统中所有数据副本保持一致。
